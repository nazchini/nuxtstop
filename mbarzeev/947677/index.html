Well, I cheated a little, since I did not completed the migration as requested, specifically this part:

// before
const container = document.getElementById('root');
ReactDOM.render(<App />, container);

// after
const container = document.getElementById('root');
const root = ReactDOM.createRoot(container);

Let‚Äôs do that and see if it fixes the error I‚Äôm getting.
My rendering code used to be like this:

const render = () => {
   ReactDOM.render(
       <Provider store={gameStore}>
           <App />
       </Provider>,
       rootElement || document.createElement('div')
   );
};

And I modified it to be compatible with React 18, like this:

const container = document.getElementById('root');
const root = ReactDOM.createRoot(container);
const render = () => {
   root.render(
       <Provider store={gameStore}>
           <App />
       </Provider>
   );
};

Let‚Äôs check if the error I had is gone‚Ä¶ It is! Awesome. The game works as expected :)

What React Testing Library suggests is to install the latest alpha version, let‚Äôs try that

Yep. running the tests now does not have any warnings. I‚Äôm not feeling that comfortable with an alpha version, but I can live with this one for now.

And so my Word-Search game is now updated with the RC version of React 18 and all seems to work well. Since the game does not have a BE involved, most of the new features of React 18 are irrelevant, but who knows‚Ä¶ they might be relevant in the future ;)
